Additional Details for TRAPShield: Real-Time APT File Theft Defense with Multi-Domain Behavioral Characterization and Transparent Decoy Redirection

TRAPShield presents an innovative defense framework against Advanced Persistent Threat (APT) file theft attacks. Its key contributions include:

  1. TRAPShield monitors and classifies sensitive file access behaviors on potentially compromised endpoints, and integrates a decoy-based deception mechanism to safeguard critical data assets and track attack activities. It establishes a minimum defense boundary and avoids heavy workload of full-cycle detection, eliminating the risk of data leakage caused by unavoidable false negatives in early-stage threat identification.
  2. We design an effective malicious sensitive file access behavior identification approach. By maintaining a limited window of detailed historical behavioral events, TRAPShield extracts multi-domain, multi-granularity features to generate threat scores for each file access attempt. Combined with an adaptive process-level identity verification mechanism, it can comprehensively identify malicious theft behaviors targeting sensitive files.
  3. We design an attacker-transparent redirection mechanism to counter malicious file access attempts. Specifically, we pre-construct a decoy file library covering diverse topics and categories. For each detected sensitive file theft behavior, TRAPShield selects a decoy counterpart for the target file and redirects access to the deceptive decoy file through interception techniques, which can effectively prevent real sensitive data from being stolen. Based on the deception strategy, we can continuously track APT attack activities without damaging any critical files, thereby reconstructing complete attack scenarios to uncover adversarial tactics and intentions.
  4. We conduct extensive evaluations three groups of datasets constructed from large-scale real-world APT reports, covering diverse APT file theft modalities and advanced attack scenarios. TRAPShield achieves F1 scores above 90.95% in detecting malicious access to sensitive files, and only applying identity authentication for less than 5% of accesses, it can achieve zero false negatives. The average detection and redirection latency is 0.125 ms, enabling real-time file protection in real-world environments. Furthermore, decoy-triggered events enable comprehensive tracking full attack chain and extracting multi-stage malware samples, facilitating further forensic analysis and threat attribution.

Datasets and artifacts of TRAPShield are available at https://osf.io/h6cyv/.

A. Preliminary Study on Quantitative Analysis of Multi-domain Behaviors

The case analyses reveal clear behavioral distinctions between benign and malicious sensitive file accesses. We leverage Analysis of Variance (ANOVA) to quantify these differences. The null hypothesis \( H_0 \) posits no significant difference exists in behavioral features between the two access categories. We assess six feature domains: ① specific file operation frequencies, ② registry operation frequencies, ③ TCP/IP network event counts, ④ specific file type access frequency and diversity, ⑤ statistical characteristics of network packet lengths, and ⑥ N-gram patterns of file I/O actions.

We perform one-way ANOVA to determine whether to reject \( H_0 \). Table avg_pvalue_by_domain presents the average $p$-values across the six feature domains. A significance threshold of \( \alpha = 0.05 \) is used, $p$-values below this indicate statistical significance. As can be seen from the table, all domains yield $p$-values well below 0.05, confirming that benign and malicious accesses exhibit statistically behavioral differences. For instance, the 4-gram pattern FileIOFileCreate_FileIORead_FileIORead_FileIORead yields a $p$-value of $2.54\mathrm{e}{-14}$, and the entropy of packet lengths yields $8.11\mathrm{e}{-70}$, both far less than 0.05. It indicates that multi-domain historical behavioral features are highly discriminative for identifying benign and malicious sensitive file accesses.

B. Implementation Details of TRAPShield

We set the length of the historical time window to 60 seconds. The $n$ value of file I/O N-gram patterns is 3 and 4. We adopt the RandomForest ensemble classifier for malicious sensitive file access behavior identification. The number of base estimators is set between 30 and 200, with a maximum tree depth of 10. These hyperparameter settings are chosen to mitigate noise and enhance the generalization in high-dimensional feature spaces. For data re-balancing, we use 3 neighbors in the ENN component and set the kind_sel to mode. We leverage the ChatGPT-4o LLM to construct the decoy file library. The weights $w_1$ and $w_2$ of the two similarity metrics for candidate decoys selection are set to 0.7 and 0.3. For attack scenario reconstruction, the number of traced parent hops is set to 3. These hyperparameter settings yield the best performance on our development sets.

C. Multi-Domain, Multi-granularity Behavioral Feature Visualization Analysis

1. Feature Importance Visualization

We further display the importance of specific features through visualization to improve the interpretability of TRAPShield for security forensic analysts. We extract feature importance scores from the RandomForest classifier trained on all five APT file theft datasets. For each attack scenario, the feature importance scores are normalized by min-max scaling, and then averaged across all datasets.

As observed, features related to registry operations including open, query, and modification behaviors at both the process and parent-process levels play a critical role in identifying sensitive file theft activities. Additionally, the file I/O behaviors related to the file opening, writing, and directory enumeration at the process and global granularities are consistently important. Furthermore, statistical features capturing process interactions with specific file types, including ini, xlsx, and ppt, are also highly indicative across multiple datasets.

2. Feature Discrimination Visualization

We further evaluate the effectiveness of the multi-domain, multi-granularity features extracted by TRAPShield through t-SNE visualization. Specifically, for each sensitive file-access behavior, we standardize the extracted 570-dimensional feature vector, reduce its dimensionality to 50 using PCA, and then project it into a two-dimensional embedding space using t-SNE.

Figures (a)–(c) illustrate the distributions of benign and malicious sensitive file access behaviors in scenarios M-1 and M-2, B-1 and B-2, and C-1, where malicious accesses are highlighted in red. The features corresponding to APT attackers’ file theft activities—including periodic manual theft, automated scheduled theft, and covert theft via process injection—are clearly separable from benign accesses in the embedding space. Malicious accesses form clustered and continuous band-shaped structures, reflecting the temporal evolution and sequential continuity of file access behaviors while maintaining clear boundaries from benign clusters. These findings demonstrate that the multi-domain and multi-granularity behavioral features extracted by TRAPShield—covering file and registry operation patterns, statistical network activity measures, and file I/O sequence characteristics—are highly effective in distinguishing malicious file theft attacks from normal access behaviors.

Figures (d)–(f) further visualize the feature distributions of different processes accessing sensitive files in datasets M-2, B-2, and C-1. Each process is color-coded. In addition to malicious file-theft processes, normal processes are also included, such as the Windows file manager (explorer.exe); file-editing applications (e.g., WINWORD.EXE, wps.exe, notepad.exe); file-synchronization processes (e.g., wpscloudsvr.exe); applications with file-transfer capabilities (e.g., mailmaster.exe, WeChat.exe, firefox.exe); and system processes (svchost.exe). These applications’ file-access behaviors exhibit strong intra-class compactness and inter-class separability, indicating that the features extracted by TRAPShield not only effectively distinguish malicious and benign sensitive file-access processes, but also capture discriminative characteristics across different application categories.

D. Runtime Overheads of Malicious Sensitive File Access Behavior Identification

The malicious sensitive file access behavior identification module of TRAPShield consists of the offline processing phase and online prediction phase. Table offline_online_time reports the runtime overheads for both phases across different APT file theft datasets. Offline operations include data pre-processing, class re-balancing, classifier training, and authentication-based threshold selection. The duration of each step ranges from seconds to minutes, with the class re-balancing stage being the most time-consuming, averaging approximately 6.5 minutes. For the online prediction phase, the average time required to profile and classify each sensitive file access behavior is below 0.26 ms across all datasets. As evaluated in Section IV.D, the average overhead of the attacker-transparent decoy file redirection module is within 0.125 ms. These results demonstrate that TRAPShield supports real-time detection and defense of sensitive file theft behaviors.

E. Further Evaluation of the Effectiveness of Decoy Files Redirection

In this section, we further evaluate the effectiveness of TRAPShield's attacker-transparent decoy file redirection module. We selected 5 popular office applications, including Word, PowerPoint, EXCEL, notepad, and Adobe, which are easily abused by attackers as injection targets. We inject the DLL into the running application processes and enable the decoy file redirection function. Then we build automated scripts to drive each application to iteratively perform file opening, reading, and closing operations on files of corresponding type, repeating this process five times for the 675 sensitive files.

The above figure shows the cumulative distribution functions (CDFs) of the interception and redirection time cost, grouped by file type and application. It can be seen that for the 5 selected office applications, over 90% of hijacking and redirection operations incur a latency below 0.2 ms. In general, TRAPShield's transparent decoy file redirection mechanism remains imperceptible to attackers and does not noticeably affect normal users performing file operations with daily office applications.

We further evaluated the runtime overheads of the minifilter-driver-based decoy file redirection mechanism, which serves as a fallback when Detours-based DLL injection into the target attack process fails (a condition that did not occur during our evaluation). The following figure shows the CDFs of redirection time cost measured for different file types. The mean latency ranges from 0.0128 to 0.0148 ms, and nearly all redirections complete within 0.2 ms. Compared with the DLL-injection-based decoy redirection, the driver-based approach exhibits significantly higher efficiency, although it introduces additional kernel–user space communication overhead. To minimize system impact while maintaining fine-grained process control and redirection reliability, TRAPShield adopts a hybrid redirection strategy that seamlessly combines both approaches.

During the endpoint behavior log collection and storage process, TRAPShield retains only a short window of recent full logs while applying simplification techniques to the larger body of historical logs. The decoy triggering-based attack tracking and reconstruction module traces simplified historical event logs to perform global association analysis. Tables log_storage_compression and log_storage_compression2 present the disk space consumption before and after simplification for each APT file theft dataset, along with the memory usage required to load the full historical logs. The evaluation environment is a Windows 10 64-bit virtual machine with 12GB of RAM and 100GB of storage. The corresponding simplification ratios are also reported. As shown, simplified storage of historical logs reduces disk usage by more than 97% and memory consumption by over 92%. In all scenarios, the full simplified logs occupy less than 200 MB of disk space, and loading the entire set of summarized historical logs for correlation analysis requires less than 300 MB of memory. These results demonstrate that TRAPShield supports efficient attack activities tracking and scenario reconstruction with modest computational resources.
